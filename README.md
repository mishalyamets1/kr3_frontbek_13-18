# KR3 Frontend + Backend (Практики 13-18)

Учебный проект по дисциплине «Фронтенд и бэкенд разработка».

В проекте реализованы:
- SPA с загрузкой контента без перезагрузки страницы.
- Локальное хранение заметок в localStorage.
- WebSocket-события через Socket.IO.
- Push-уведомления через Service Worker и Web Push.
- Напоминания с отложенной отправкой (практика 17).
- Действие в уведомлении: «Отложить на 5 минут».

## Стек

- Frontend: HTML, CSS (chota), JavaScript
- Backend: Node.js, Express, Socket.IO
- Push: web-push + Service Worker API

## Структура проекта

- app.js — клиентская логика, SPA, формы заметок и напоминаний.
- server.js — сервер, Socket.IO, хранение подписок и таймеров напоминаний.
- sw.js — Service Worker, кэширование, push и обработка кликов по уведомлениям.
- content/home.html — UI заметок и формы напоминаний.
- content/about.html — страница «О проекте».
- index.html — основная страница приложения.

## Требования

- Node.js 18+ (рекомендуется LTS).
- Современный браузер с поддержкой Service Worker и Push API.

## Как проверить функционал

### 1. Обычные заметки

1. Нажмите «Включить уведомления».
2. Добавьте заметку через форму «Добавить заметку».
3. Должно прийти push-уведомление «Новая заметка».

### 2. Напоминания (Практика 17)

1. В форме «Добавить с напоминанием» введите текст и будущую дату/время.
2. После наступления времени должно прийти push-уведомление «!!! Напоминание».
3. В уведомлении доступно действие «Отложить на 5 минут» (в поддерживаемых браузерах/ОС).

## Как работает напоминание

- Клиент отправляет событие newReminder с id, text, reminderTime.
- Сервер создает таймер setTimeout и хранит его в Map reminders.
- При срабатывании таймера сервер отправляет push с reminderId.
- Service Worker добавляет action-кнопку для reminderId и обрабатывает snooze.
- При snooze вызывается POST /snooze, сервер переносит уведомление на 5 минут.
